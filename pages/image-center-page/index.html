Caption Categories

自动生成标签流程配置

⚙️ 自动标签生成设定

设置用于图库图片自动特征标签生成的图片转述模型。

默认继承 AstrBot 系统活动配置文件中的“默认图片转述模型”设置；保存后会同步写入系统配置。

需要选择一个支持图片理解的可用图片转述模型。

✍️ 生成标签类别设定

控制插件为图片自动赋予特征标签时，优先基于哪些类别来生成。

可勾选的预设类别：图像类别 (是照片还是表情包)、人物特征、身材、情绪、动作、图片中的文本。

可输入多个自定义类别，各个类别之间可以用空格、逗号或顿号隔开。

注意：该操作会清空图库中原先的特征标签 (公用标签除外)

Proactive Emoji Reply

对话中主动发送表情包

📤️ 对话中主动发送表情包

调用 LLM 对 bot 回复用户的内容进行语义分析，从图库中选择并发送符合语境的表情包。

当 bot 的对话由 LLM 生成，且概率命中时，启动语义情感分析 + 表情包发送流程。

继承 AstrBot 系统配置中的模型提供商；留空或不可用时，继承当前会话的模型选择策略。建议选择实时性更强的模型，如 gpt-5.4-mini 或 deepseek-v4-flash。

开启后，只从带“表情包”特征标签的图库图像中筛选。

开启后，表情包和对话在同一次对话中一起发送；关闭后，对话遵循原先的 LLM 普通对话发送流程，表情包会以非 LLM 输出格式独立发送。关闭此选项可避免与其他插件产生兼容性冲突。

取值范围 0-1。只有普通 LLM 回复概率命中时，才启动语义和情感分析 + 表情包选择流程。

Auto Image Collection

自动收集群友发的图

📩 自动收集群友发的图

在指定 QQ 群中收集图片。图片先进入 [待筛选图片池]，用户需要选择哪些图像加入 [固化图像库]，入库后才会自动打标签、被检索。

开启后，插件会在指定 QQ 群中自动收集图片，放入 [待筛选图片池]。

(建议关闭) 关闭时，用户可手动指定 [待筛选图片池] 中哪些图片存入 [固化图像库]。开启后，图像将实时自动入库。

开启后，[固化图像库] 中的图片，会和手动上传图像一起，作为“为用户寻找表情包”和“对话中主动发送表情包”的检索候选项。

每行一个 QQ 群号。机器人只在这些群聊中自动收集图片。

开启后，会记录被 [待筛选图片池] 手动丢弃的图片信息；再次见到相同图片时直接跳过。

设置为 -1 表示无上限。

User Image Search

为用户寻找表情包

🔍️ 寻找表情包功能开关

用户唤醒机器人后，如果对话中存在下列关键词，则在图库中检索满足要求的表情包。

关闭后，即使用户对话包含下方关键词，也不会触发机器人为用户检索并发送图库图片；不会影响“对话中主动发送表情包”功能。

用户唤醒机器人后，消息中包含任一关键词才会进入本插件流程。注意：自动标签生成必须在 AstrBot 的 [系统配置] 中，配置 [默认图片转述模型]。

🎤 寻找表情包后对用户的回复

这里集中管理命中图片、未命中图片和图库为空时的回复方式。

关闭后会继续一次 AstrBot 原 LLM 回复流程，并提示模型告知用户已经找到图片。

支持 {filename}、{tags}、{need}、{reason} 占位符。
